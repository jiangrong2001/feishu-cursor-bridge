还在电脑和手机之间来回切、把需求复制到 Cursor 里？

分享一个我最近在用的开源小项目：feishu-cursor-bridge（飞书 ↔ Cursor 桥接）。

它能干嘛？
• 本机跑一个轻量 Node 服务，用飞书 WebSocket 长连接收消息——不用搭公网 Webhook，家里/公司内网都能用。
• 收到你在飞书里发给机器人的话，自动 spawn 官方 Cursor Agent CLI（headless），在你指定的工作区里读代码、改文件、跑终端命令。
• 回答通过 lark-cli 或桥接 API 发回飞书，尽量做到不空白、不重复刷屏，体验接近一问一答。

适合谁？
• 想用飞书当「远程遥控器」、让本机 AI 干活的开发/技术负责人。
• 小团队想统一入口：人在外，手机飞书 @ 机器人，家里电脑上的 Agent 照样跑。

不是什么？
不是飞书云里托管的模型，算力和代码都在你自己机器上，和本机跑 agent 能力一致。

怎么玩？
开源在 GitHub：https://github.com/jiangrong2001/feishu-cursor-bridge
文档里有飞书长连接怎么配、agent login、环境变量、安全白名单、调试日志等，按 README 一步步就能起服务。
